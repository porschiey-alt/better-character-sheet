## Summary

The `_prepareContext()` method in `BetterCharacterSheet.ts` is ~630 lines and builds all template data inline. Much of this logic could be extracted into focused helper functions in `data-transforms.ts` (or new files), matching the pattern already used for abilities, saves, skills, inventory, and encumbrance.

## Proposed Extractions

The following data builders are good candidates:

- **`buildAttacks(actor, spellcasting, levelLabels)`** — Currently lines ~205–292. Builds unified attack list from equipped weapons + attack/save spells.
- **`buildActionFeatures(actor)`** — Currently lines ~295–420. Builds feat items with activation costs, expanding per-activity for multi-activity feats.
- **`buildSpellsByActivation(spellItems, levelLabels)`** — Currently lines ~422–454. Categorizes spells into bonus/reaction/other/ritual for the Actions tab.
- **`buildSpellsByLevel(spellItems, levelLabels)`** — Currently lines ~457–522. Groups spells by level with cast time, range, components, etc.
- **`buildFeatureGroups(actor)`** — Currently lines ~535–627. Groups feats by type with descriptions, pips, and sub-actions.
- **`buildSpellcastingInfo(classItems, system)`** — Currently lines ~158–196. Computes spell attack, DC, and prepared caster detection.

## Benefits

- `_prepareContext()` becomes a ~50-line orchestrator
- Each builder can be unit tested independently (like `buildAbilities` and `buildSkills` already are in `data-transforms.test.ts`)
- Easier to reason about data flow for individual UI sections
